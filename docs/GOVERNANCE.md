# Governance

## Current model
- Governance updates are submitted by a privileged governance address
- Updates are applied via `gov_update` transactions

## Recommended policy
- Maintain a public list of parameters and changes
- Require multi-signature approval for governance wallet
- Announce changes and allow review time
- Keep an upgrade calendar and version policy

## Suggested parameters to govern
- target_block_time
- max_block_gas
- base_gas_price
- min_relay_fee
- max_mempool_bytes
- finality_depth
