# Incident Response

## Severity levels
- Sev 0: chain halted or critical loss
- Sev 1: consensus divergence or major exploit
- Sev 2: degraded performance or partial outage
- Sev 3: minor bug or UI issue

## Immediate steps
- Triage and identify impact
- Contain by pausing mining or isolating nodes
- Communicate status to users
- Gather evidence and logs

## Follow-up
- Root cause analysis
- Patch and redeploy
- Post-incident report
- Update tests and runbooks
