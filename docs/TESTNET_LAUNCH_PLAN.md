# Testnet Launch Plan

## Phase 0 - Local validation
- Run the demo script and tests
- Validate RPC, indexer, and UI locally
- Confirm snapshots and restore on a fresh data dir

## Phase 1 - Private testnet
- Deploy 3 to 5 nodes on separate hosts
- Configure seed peers and peer discovery
- Enable monitoring and basic alerts
- Run load tests and spam tests
- Document known issues and fixes

## Phase 2 - Public testnet
- Publish seed node list and explorer URL
- Launch a faucet for test tokens
- Track uptime and block time targets
- Open a bug bounty
- Collect feedback from external testers

## Phase 3 - Mainnet candidate
- Freeze chain parameters and tokenomics
- Run testnet for 4 to 12 weeks with no critical issues
- Complete independent security audit
- Finalize governance and upgrade policy
- Prepare mainnet launch checklist
